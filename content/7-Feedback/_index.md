> This section is intended for sharing your experience after completing the Smart Parking workshop. Your feedback will help improve future versions of the workshop and provide valuable insights for other learners.

## Overall Evaluation

### 1. Workshop Content

The workshop provides a structured learning path, guiding participants through the process of deploying a Smart Parking System on AWS. The step-by-step instructions make it easier to understand how different AWS services work together in a real-world application.

### 2. Learning Experience

The hands-on activities helped reinforce theoretical knowledge by allowing participants to deploy and configure AWS resources such as Amazon EC2, Amazon RDS, Amazon S3, and Amazon CloudWatch. This practical experience improved understanding of cloud architecture and application deployment.

### 3. Technical Skills Gained

After completing the workshop, participants will gain experience in:

- Deploying applications on Amazon EC2
- Configuring Amazon RDS databases
- Managing storage with Amazon S3
- Monitoring applications using Amazon CloudWatch
- Configuring networking and security on AWS
- Understanding cloud-based application deployment

### 4. Challenges Encountered

During the deployment process, participants may encounter issues such as:

- Security Group configuration
- Database connection errors
- Application deployment failures
- EC2 connectivity problems

These challenges provide valuable opportunities to strengthen troubleshooting and problem-solving skills.

### 5. Overall Impression

The Smart Parking workshop demonstrates how multiple AWS services can be integrated into a complete cloud-native solution. It provides practical experience in deploying, configuring, and managing cloud infrastructure.

## Additional Questions

### 1. Which part of the workshop did you find the most useful?

**Answer:**  
The most useful part of the workshop was deploying the Smart Parking application on Amazon EC2. It provided practical experience in launching an EC2 instance, configuring the environment, deploying the application, and verifying that the system was running successfully.

### 2. Which AWS service was the most challenging to configure?

**Answer:**  
Amazon RDS and Security Group configuration were the most challenging parts. Establishing a successful connection between the application and the database required careful configuration of networking and access permissions. However, completing these tasks helped improve my understanding of AWS networking and database management.

### 3. What additional topics would you like to see included in future workshops?

**Answer:**  
I would like future workshops to include more advanced topics such as deploying applications with Amazon ECS, implementing CI/CD pipelines using GitHub Actions or AWS CodePipeline, configuring HTTPS with AWS Certificate Manager and Application Load Balancer, and monitoring applications with CloudWatch dashboards and alarms.
