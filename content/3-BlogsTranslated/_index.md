This section introduces the translated blogs related to the **Smart Parking System on AWS**. Each blog focuses on a different part of the cloud architecture, explaining the AWS services, design decisions, and implementation techniques used to build a scalable, secure, and highly available smart parking platform.

---

### [Blog 1 - Introduction to Smart Parking System Architecture](3.1-Blog1/)

This blog provides an overview of the Smart Parking System architecture deployed on AWS. It explains the overall cloud-native design, including the Presentation Layer, Application Layer, Integration Layer, Data Layer, IoT Layer, and Security & Monitoring Layer. The article also introduces the AWS services used to improve scalability, availability, security, and system performance.

---

### [Blog 2 - Building Microservices with Amazon ECS](3.2-Blog2/)

This blog focuses on the Application Layer of the Smart Parking System. It explains how Amazon ECS is used to deploy backend services following the Microservices Architecture. The article introduces the responsibilities of each service, including Auth Service, User Service, Parking Service, Booking Service, Payment Service, and Notification Service, while demonstrating how ECS improves deployment, scalability, and maintenance.

---

### [Blog 3 - Event-Driven Architecture with AWS Integration Services](3.3-Blog3/)

This blog introduces the Event-Driven Architecture used in the Smart Parking System. It explains how Amazon EventBridge, AWS Lambda, Amazon SQS, Amazon SNS, and Amazon SES work together to process asynchronous events such as booking confirmation, payment processing, QR Code generation, and notification delivery. The article highlights the benefits of loose coupling, scalability, and fault tolerance.
