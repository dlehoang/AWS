### Week 5 Objectives:

* Understand cloud migration methodologies and strategies (The 6 Rs) to transition workloads to AWS.
* Learn how to discover existing on-premises infrastructure and perform secure database migrations.

### Tasks to be carried out this week:
| Day | Task | Start Date | Completion Date | Reference Material |
| --- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------- | --------------- | ----------------------------------------- |
| 2   | - Learn about AWS Cloud Migration concepts, frameworks, and the 6 Rs migration strategies (Rehost, Replatform, Refactor, etc.). | 18/05/2026  | 18/05/2026      | <https://cloudjourney.awsstudygroup.com/> |
| 3   | - Study AWS Application Discovery Service to inventory and gather performance data from on-premises servers. | 19/05/2026 | 19/05/2026      | <https://cloudjourney.awsstudygroup.com/> |
| 4   | - Learn about AWS Database Migration Service (DMS) and AWS Schema Conversion Tool (SCT) for database migration. | 20/05/2026 | 20/05/2026      | <https://cloudjourney.awsstudygroup.com/> |
| 5   | - Explore continuous data replication methods and minimizing downtime during database transitions. | 21/05/2026 | 21/05/2026      | <https://cloudjourney.awsstudygroup.com/> |
| 6   | - Practice setting up an AWS DMS replication instance and configuring basic source-to-target database migration tasks. | 22/05/2026 | 22/05/2026      | <https://cloudjourney.awsstudygroup.com/> |

### Week 5 Achievements:

Mastered the 6 Rs migration pathways to analyze and choose the appropriate strategy for moving various enterprise workloads to the cloud.

Understood how to effectively inventory on-premises server environments and map system dependencies using AWS Application Discovery Service.

Grasped the mechanism of homogeneous and heterogeneous database migrations using AWS SCT and AWS DMS.

Successfully initialized an AWS DMS replication instance, configured source/target endpoint connections, and monitored data migration tasks.
